# Phase quick Plan 260330-i45: Add ability to unmark detected cells by clicking Summary

Click-to-toggle removal of algo-detected cells with live counter updates and batch persistence via centroids + removed_indices stored in manifest JSON.

## What Was Built

Scientists can now click any red numbered circle on the annotated image to remove it (counter drops); clicking the same spot again re-marks it (counter rises). The removed state survives batch save/reload. Re-running analysis resets all removals to start fresh.

## Key Changes

**analysis_core.py**
- `process_image` now returns `(viz, count, centroids_list)` — centroids_list is `[(cx, cy), ...]` for every accepted cell, including watershed sub-cell global coordinates
- `optimize_parameters` updated to unpack the 3-tuple with `_, count, _`

**workers/analysis_worker.py**
- `AnalysisSignals.image_done` changed from `Signal(str, object, int)` to `Signal(str, object, int, object)` to carry centroids
- `AnalysisWorker.run()` unpacks and emits centroids with each `image_done` signal

**ui/main_window.py**
- `_on_image_done(filename, annotated_rgb, count, centroids)` — stores `algo_centroids` and resets `removed_indices = []`
- `_on_reanalyze_image_done(filename, annotated_rgb, count, centroids)` — same, also resets removed_indices (stale index safety)
- `_load_batch_from_path` — populates `algo_centroids` and `removed_indices` from manifest on load
- `_redraw_annotated` — rewritten to draw from `original_rgb`, skip removed indices, renumber active cells sequentially, then draw manual marks on top
- `_on_annotated_click` — rewritten with 3-priority hit test: algo toggle > manual remove > add new manual

**batch_manager.py**
- `save_batch` — manifest now includes `algo_centroids` and `removed_indices` per image; `cell_count` uses active count
- `update_manifest` — same active-count formula and new fields persisted
- `load_batch` — normalizes `algo_centroids` to list of tuples and `removed_indices` to list of ints on load
