# Quick Task 260330-i45: add ability to unmark detected cells by clicking marks with counter updates and batch persistence - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Task Boundary

Allow users to unmark detected cells by clicking on the drawn circle marks in the UI. The cell counter should update immediately. When saving or loading a batch, unmarked cells must be excluded from the count and annotation.

</domain>

<decisions>
## Implementation Decisions

### Click Target Size
- Hit area = same as the drawn circle radius — intuitive, matches the visual

### Visual Treatment of Unmarked Cells
- Unmarked cells disappear entirely from the canvas
- Counter drops immediately on click

### Re-marking Capability
- Full toggle: click an active mark to unmark it, click the spot again to re-mark
- Marks should be re-drawable on canvas (or a show/hide toggle mechanism)

### Claude's Discretion
- How unmarked cell state is persisted in the batch JSON (e.g., a `removed_indices` list or a per-cell `active` flag)
- Whether the toggle re-mark works by showing a ghost/invisible hit area on the original detected position, or by some other mechanism

</decisions>

<specifics>
## Specific Ideas

- Marks are currently drawn as red circles with numbers on a canvas overlay
- Toggling should not re-run the detection algorithm — only hide/show already-detected cells
- Batch save/load must store which cells were manually removed so state is preserved across sessions

</specifics>
